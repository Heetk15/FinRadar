# ADR-004: Feature Flags for Zero-Downtime Migration

## Context
Migrating FinRadar from a synchronous request-driven architecture (V1) to a scheduled background ingestion architecture (V2) carries risk. The new database tables, scheduler, and API paths must be validated in production before completely abandoning the legacy system.

## Decision
We implemented a feature flag: `USE_V2_READ_PATH` (boolean) inside our `.env` configuration.

## Rationale
- **Safe Rollout**: By wrapping the read paths in an `if USE_V2_READ_PATH:` block, we deploy the V2 background ingestion pipeline to start collecting data (shadow mode) without affecting user traffic.
- **Instant Rollback**: If a bug is discovered in the V2 data model or query execution plans, rolling back is as simple as toggling `USE_V2_READ_PATH=false` and restarting the container. It avoids complex `git revert` operations or database down-migrations.
- **Interview & Placement Value**: Demonstrates maturity in deploying features to production safely, a key requirement for modern SaaS engineering teams.

## Consequences
- Requires maintaining two parallel execution paths in the codebase temporarily.
- Necessitates a "Cleanup Phase" (Phase 4D-C) later to delete the V1 legacy code once V2 is fully verified.
